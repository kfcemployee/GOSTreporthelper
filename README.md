### О проекте
GOST Report Helper - это мощный инструмент для автоматического форматирования документов в соответствии с требованиями ГОСТ. 
Программа преобразует обычные Word документы и Markdown файлы в правильно оформленные документы.


## Предподготовка файла
1. Программа работает только с файлами, которые изначально были сохранены как .docx (всилу особенностей docx формата). Перевести любой файл в DOCX можно очень просто:
   - открываете исходный файл в любом тектовом редакторе на вашем устройстве
   - нажимаете "Файл -> Сохранить как" или комбинацию клавиш `Ctrl + Shift + S`
   - в открывшемся окне в после "Тип файла" выбираете "документ Word 2010-365 (*.docx) и вводите имя файла:
   <img width="1436" height="149" alt="image" src="https://github.com/user-attachments/assets/b0a6f26f-da39-47f6-9135-a4d44d8d0e77" />

2. Для более точного распознавания вы можете явно указать стили текста (пометить заголовки, списки и тд). Для этого выделите текст и установите для него подходящий стиль.
3. Проверьте, не указаны ли у вас ошибочные стили.
   

## Быстрый старт
#### Способ 1: перетаскивание.
- Скачайте GOSTreporthelper.exe
- Перетащите файл .docx или .txt на исполняемый файл
- Готово! Результат сохранится рядом с исходным файлом

#### Способ 2: командная строка.
``` bash
# Базовое использование
GOSTFormatter.exe документ.docx

# С указанием выходного файла
GOSTFormatter.exe текст.txt -o отчет.docx

# Показать справку
GOSTFormatter.exe --help
```


## Установка
#### Вариант 1: готовая версия.
  - Скачайте готовую сборку с Github Releases (<a href="https://github.com/kfcemployee/GOSTreporthelper/releases/download/v1/GOSTreporthelper.exe">Файл</a>)
  
#### Вариант 2: установка исходного кода.
Перед установкой убедитесь, что у вас скачан Python версии 3.7+. если нет - скачайте с официального сайта или воспользуйтесь вариантом 1.
  Руководство по установке:
  ``` bash
  # Клонировать репозиторий
  git clone https://github.com/kfcemployee/GOSTreporthelper.git
  cd GOSTreporthelper
  
  # Установить зависимости
  pip install -r requirements.txt
  
  # Использовать через Python
  python cli.py документ.docx
  ```
Вызовите `python cli.py` или `python cli.py -h` для справки.


## Работа с готовым файлом.
Вы можете форматировать уже готовый файл, согласно ГОСТ, применяя к нему уже готовые стили. Для этого:
- Откройте документ в любом текстовом редакторе, который корректно работает к docx
- Выделите текст, который хотите форматировать
- Примените к нему подходящий стиль, названия стилей начинаются с префикса ГОСТ и совпадают с названиями встроенных стилей.
  

## Изменение существующих параметров формата
Этот раздел будет дорабатываться. функционал, представленный в данном разделе, будет улучшен и ручное изменение параметров форматирования будет удобнее))
1. Установите исходный код
2. Перейдите к файлу styles_configs/styles_config.py
3. Здесь см. функцию conf_styles класса StyleManager. эта функция возвращает словарь из всех стилей. Вы можете поменять значение в конструкторе классов на нужное вам.
(Вы также можете добавить новый стиль, воспользовавшись конструктором).
